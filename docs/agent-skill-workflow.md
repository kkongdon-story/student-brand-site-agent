# 에이전트 스킬 기반 작업 흐름

이 에이전트는 `personal-brand-site`라는 중심 스킬 하나로 시작합니다. 안에서는 여러 보조 스킬을 필요할 때 꺼내 쓰는 방식입니다.

## 전체 흐름

1. 브랜드 재료 정리
2. 레퍼런스 증거 정리
3. 프롬프트팩 만들기
4. 이미지 방향 잡기
5. `DESIGN.md` 제작
6. HTML/CSS/JS 홈페이지 제작
7. 브라우저 QA
8. GitHub/Vercel 배포 준비

## 1단계: 브랜드 재료 정리

목표는 “나는 누구이고, 누구를 돕고, 방문자가 무엇을 하게 만들 것인가”를 정하는 것입니다.

결과물:

- `brand-materials.md`

핵심 질문:

- 누구를 돕나요?
- 무엇을 제공하나요?
- 사람들이 왜 나를 믿어야 하나요?
- 방문자가 눌러야 할 버튼은 무엇인가요?

## 2단계: 레퍼런스 증거 정리

목표는 마음에 드는 사이트를 그냥 복붙하는 것이 아니라, AI가 이해할 수 있는 디자인 자료로 바꾸는 것입니다.

결과물:

- `reference-dna.md`

사용하는 핵심 스킬:

- `stitched-full-page-capture`: 레퍼런스 사이트를 화면별로 캡처
- `html-to-interaction-prompts`: 마음에 드는 움직임을 프롬프트로 추출
- `design-first-ui-prompting`: 감각을 디자인 지시문으로 변환
- `high-end-visual-design`: 흔한 AI 템플릿 느낌을 막는 규칙 적용

## 3단계: 프롬프트팩 만들기

목표는 한 번 쓰고 사라지는 대화가 아니라, 계속 재사용할 수 있는 지시문을 만드는 것입니다.

결과물:

- `prompt-pack.md`

포함 내용:

- 전체 디자인 슈퍼 프롬프트
- 섹션별 프롬프트
- hover/click/scroll 인터랙션 프롬프트
- 이미지 생성 프롬프트
- 수정 요청 프롬프트
- 하면 안 되는 디자인을 막는 negative prompt

## 4단계: 이미지 방향 잡기

목표는 홈페이지에 들어갈 캐릭터, 인물, 오브젝트, OG 이미지의 방향을 정하는 것입니다.

결과물:

- `image-brief.md`

기본 방식:

- 로컬 `imagen`, `imagegen`, `openai-image` CLI를 먼저 사용합니다.
- API가 없으면 실제 이미지를 만든 척하지 않고, 생성해야 할 이미지 지시문과 파일 위치를 남깁니다.

## 5단계: DESIGN.md 제작

목표는 Codex가 그대로 따라 만들 수 있는 제작 계약서를 만드는 것입니다.

결과물:

- `DESIGN.md`

`DESIGN.md`에는 아래가 들어가야 합니다.

- 홈페이지 목적
- 브랜드 재료
- Hero 카피
- 레퍼런스 DNA
- 레퍼런스 증거팩
- 비주얼 방향
- 디자인 시스템 제약
- 프롬프트팩 요약
- 이미지 자산
- 색상/타이포/레이아웃
- 섹션 상세
- 모션과 인터랙션
- 반응형 규칙
- SEO/OG/GEO
- QA 기준

## 6단계: 홈페이지 제작

목표는 초보자도 나중에 수정할 수 있는 정적 홈페이지를 만드는 것입니다.

결과물:

```text
site/
  index.html
  styles.css
  script.js
  robots.txt
  sitemap.xml
  llms.txt
  README.md
```

사용하는 핵심 스킬:

- `landing-page`: 한 대상, 한 제안, 한 행동으로 구조 정리
- `animation-on-scroll`: 기본 스크롤 등장 효과
- `cinematic-scroll-storytelling`: 고급 스크롤텔링이 필요할 때만 사용

## 7단계: 브라우저 QA

목표는 코드가 있다는 사실이 아니라, 실제 화면에서 작동하는지 확인하는 것입니다.

확인 항목:

- 데스크톱 스크린샷
- 모바일 스크린샷
- hover 반응
- click 반응
- scroll 반응
- 모바일 가로 스크롤 없음
- SEO/OG/GEO 기본 메타데이터
- `robots.txt`, `sitemap.xml`, `llms.txt`

사용하는 핵심 스킬:

- `optimize-web-animations`: 움직임이 너무 무겁거나 모바일에서 버벅이지 않는지 확인

## 8단계: 배포 준비

목표는 내 컴퓨터 안의 파일을 실제 링크로 만드는 것입니다.

흐름:

1. GitHub에 코드 올리기
2. Vercel에 GitHub 저장소 연결하기
3. 배포 URL 확인하기
4. 카카오톡, 스레드, 인스타, 블로그에 공유하기

## 핵심 원칙

AI에게 바로 “홈페이지 만들어줘”라고 하지 않습니다.

먼저 내 브랜드를 정리하고, 레퍼런스를 증거로 바꾸고, 프롬프트팩과 `DESIGN.md`를 만든 뒤, 그 기준으로 제작합니다. 이 순서가 있어야 결과물이 흔한 AI 템플릿이 아니라 나만의 브랜드 홈페이지에 가까워집니다.
