# Stage 3 - DESIGN.md Contract

Goal: compile brand materials, reference evidence, prompt assets, and image direction into an implementation contract.

`DESIGN.md` is not a mood memo. It is the source of truth for the website builder.

## Required Sections

1. Website Purpose
2. Brand Materials
3. Hero Copy
4. Reference DNA
5. Reference Evidence Pack
6. Visual Direction
7. Design System Constraints
8. Prompt Pack
9. Image Assets
10. Color System
11. Typography
12. Layout
13. Section Detail
14. Motion and Interaction
15. Components
16. Responsive Rules
17. SEO / OG / GEO
18. Implementation Files
19. QA Acceptance Criteria
20. Must Not Drift

## Contract Rules

- Every abstract phrase must be translated into visible decisions.
- Every section must say what to build and what to avoid.
- The reference lane must include source, capture or screenshot plan, bring, avoid, and AI-readable material.
- The prompt lane must include a design-first super prompt, section prompts, interaction prompts, and a negative prompt.
- The design system lane must name visual archetype, layout archetype, type scale, spacing rhythm, color/material, component rules, motion rules, and anti-generic guardrails.
- The image lane must name exact asset filenames or exact pending prompts.
- The motion lane must name hover, click, and scroll behavior.
- The QA lane must be testable by a browser or CLI script.
